Design Patterns

In the development of [Game Title], we made use of a number of design patterns that we learned about in class. These patterns helped us to [list of benefits of using design patterns].

Here are some specific examples of how we used design patterns in our game:

  • Singleton Pattern: We implemented this pattern so that there would be only one instance of the Leaderboard and Player at a time. This ensured that we would always access the correct object and perserve changes across game states.
  • Observer Pattern: This pattern was implemented so that objects can observe changes in the Player or RoomViewModel classes. For example, enemies that observe the player are notified whenever the player moves, prompting them to move as well.
  • Strategy Pattern: The strategy pattern was used to create similar instances with similar but different functionalities. There are 2 movement strategies for the player based on keyboard or touch input, and 4 strategies for the different types of enemy movement.
  • Factory Pattern: The factory pattern was used to easily instantiate different enemy types based on a designated input variable.
  • Decorator Pattern: We used the decorator pattern to implement the different functionalities of powerups to ensure they would only affect the player.
